Controle de versão usando Git e GitHub

Inicializando o projeto ``` $ mkdir ideal-adventure $ cd ideal-adventure $ git init ``` Configurando o usuário ``` $ git config user.name "Gabriele Luz" $ git config user.email gabrieleluz@github.com ``` _O comando git config --list lista as configurações setadas no seu repositório_

Rastreando arquivos

#### Git status Mostra o status do diretório de trabalho ``` $ git status ```
#### Git add Informa ao git que as mudanças no(s) arquivo(s) indicado(s) devem ser rastreadas. ``` $ git add < arquivo > ```

Exercício

  • Verifique o status do diretório com o comando `$ git status`
  • Informe ao git para rastrear as alterações de todos os qrquivos
  • [fragment] Você pode fazer isso adicionando todos os arquivos manualmente `$ git add < arquivo >` ou usar `$ git add .` para adicionar todos os arquivos
## Gravando alterações no repositório #### Git commit Grava mudanças no repositório ``` $ git commit -m "mensagem" ``` *É importante que a mensagem do commit diga respeito ao que foi feito na alteração.*
