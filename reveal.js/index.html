Controle de versão usando Git e GitHub

Inicializando o projeto ``` $ mkdir ideal-adventure $ cd ideal-adventure $ git init ``` Configurando o usuário ``` $ git config user.name "Gabriele Luz" $ git config user.email gabrieleluz@github.com ``` _O comando git config --list lista as configurações setadas no seu repositório_

Rastreando arquivos

Git status Mostra o status do diretório de trabalho ``` $ git status ```
Git add Informa ao git que as mudanças no(s) arquivo(s) indicado(s) devem ser rastreadas. ``` $ git add < arquivo > ```

## Gravando alterações no repositório Git commit Grava mudanças no repositório ``` $ git commit -m "mensagem" ``` *É importante que a mensagem do commit diga respeito ao que foi feito na alteração.*

## Trabalhando com servidor remoto ### Adicionando ao servidor remoto ``` $ git remote add origin https://github.com/< usuario >/< repo > ``` ### Enviando alterações para o servidor remoto ``` $ git push origin < nome-da-branch > ```
## Colaborando ### Baixando um novo repositório Git clone Clona um repositório numa nova pasta. ``` $ git clone http://github.com/< usuario >/< repo > ```

## Baixando novas alterações ### Git pull Baixa alterações realizadas no repositório remoto. ``` $ git pull < remoto > < branch > ``` Em breve entenderemos o funcionamento de branches. Por enquanto, utilize o comando ```$ git pull origin master``` para baixar alterações.

Branches

Linhas de *desenvolvimento* paralelas. São comumente utilizadas para: - Desenvolver novas funcionalidades; - Correções de bugs; - Organizar o repositório logicamente de acordo com o que está sendo desenvolvido.
Criando uma nova branch ``` $ git branch < nome-da-branch > ``` Listando branches ``` $ git branch --list ``` Acessando branches ``` $ git checkout < nome-da-branch > ``` Deletando Branches ``` $ git branch -D < nome-da-branch > ```
### Master - Linha de desenvolvimento principal - Criada automaticamente ao inicializar o git
Enviando alterações a partir de uma nova branch ``` $ git push origin < nome-da-branch > ```
